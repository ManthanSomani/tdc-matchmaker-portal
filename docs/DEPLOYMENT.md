# Deployment runbook

## Target topology

- Vercel: `client/` as the project root
- Render: Express service from `server/`
- Hugging Face Docker Space (or a second Render service): FastAPI sentence-transformer API

## 1. ML service

Deploy `server/` with `Dockerfile.nlp`, expose port 7860, and set `CORS_ORIGINS` to the Express origin. Record its HTTPS origin.

## 2. Express API

Create/sync the root `render.yaml`. Set every `sync: false` value in the Render dashboard: frontend CORS origin, bcrypt admin hash, ML origin, and optional Groq key. `JWT_SECRET` is generated by the Blueprint. Verify `/health`, login, directory paging, match fallback/source metadata, and intro drafting.

## 3. Frontend

Import the GitHub repository in Vercel and set Root Directory to `client`. Set `VITE_API_URL=https://<api-host>/api` for preview and production, then deploy. Add the final Vercel URL to `CORS_ORIGINS` on both backend services.

## Release verification

Run the automated checks, deploy a preview, and manually verify sign-in, search, page navigation, scoring explanations, AI/template source behavior, sign-out, mobile layout, and browser console/network errors. Never place server secrets in `VITE_*` variables.

Free services can cold-start. Keep the fallback visible and do not claim production uptime.
